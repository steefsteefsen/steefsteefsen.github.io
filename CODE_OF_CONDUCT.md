# Code of Conduct

This project follows the same conduct rules that the
[ethics page](ethics/index.html) on the site itself lays out.

## Direct & quiet

Two words. They are the tribe codename („direkt & heimlich") and
they are the conduct here too.

**Direkt** — say things by name. If a card is wrong, say it's wrong.
If a name should be removed, say so. If you disagree with a decision,
disagree out loud, with reasons. No passive-aggressive issues, no
*"just wondering"* when what you mean is *"this is broken"*.

**Heimlich** — don't perform. No virtue-signalling in PRs, no
*"as a person who deeply values X"* preambles, no badges, no
attention-economy moves. The work is the work.

## What's not OK

- Doxxing anyone, in this repo or anywhere else. The site has a
  Tier-2 privacy rule for a reason — see CLAUDE.md.
- Adding real names without explicit consent.
- Performative outrage. If something needs criticism, criticise
  the thing, not the contributor.
- Crypto-dust, "AI-generated" PR-spam, low-effort issues to inflate
  GitHub-style metrics.

## Disagreements

Disagree directly. Stefan replies to direct mail. Public PR
discussion is welcome but should stay technical.

If conduct issues arise that the contributors can't resolve
amongst themselves: <steef@huellinghorst.info> with subject
*„Conduct"*.
