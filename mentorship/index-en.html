Mentorship

Learn by building something real

If I mentor you, this is what it looks like. Open to anyone who shares the values — no application, no CV, no certificate at the end. Just code that lives.

How this works

You fork this site. You install Claude Code with your own account. You take the first PR from the list below and ship it. Once it's merged, the next one. After four PRs we talk about what you want to work on next.

No sessions, no calls, no curriculum. You work, I review, we talk when it makes sense.

Before you start

  1. Read the philosophy. If something doesn't sit right, say it now — that saves us both time.
  2. Get Claude Code: npm install -g @anthropic-ai/claude-code, then claude in the repo.
  3. Fork the repo on GitLab.
  4. Read CLAUDE.md in the repo root. That's the user manual.

The first four PRs

In this order. Don't start PR #2 before PR #1 is merged.

PR #1 — A tool card in „Built With"

Pick a tool you actually use. Add a tool-card in index.html. Add the i18n key to all 17 language blocks in i18n.js. Get the official logo. Tests green. Push.

Teaches: HTML, i18n, asset sourcing, the test loop. The minimal full cycle.

PR #2 — One support card

An artist, a local business, or a cause that matters to you. Just one. With a footnote if you cite external facts.

Teaches: the footnote system. Voice and tone.

PR #3 — One language you actually speak

Pick a language from i18n.js that you really know. Replace the EN fallbacks with real translations. One language only. Don't guess.

Teaches: scope discipline. A big visible win.

PR #4 — A blog post (DE+EN)

A topic you actually have something to say about. Two HTML files, an entry in posts.json, regenerate the feed. If you make factual claims, check them against primary sources.

Teaches: end-to-end ownership.

What „ready" means

  • Tests are green (cd tests && npx jest)
  • You can say in one sentence what changed and why
  • You read your own diff before you pushed
  • If UI: you opened it in the browser at 375px and 1280px

That's it. No checklist theatre.

When you're stuck

  • <10 min stuck: keep going.
  • 10–30 min: ask Claude differently. Reformulate the problem.
  • >30 min: write to me. What you wanted, what you tried, what happened. No apology. That's the deal.
