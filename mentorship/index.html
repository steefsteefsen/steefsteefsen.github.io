Mentorship

Lernen, indem du etwas Echtes baust

Wenn ich dich mentore, sieht es so aus. Offen für alle, die die Werte teilen — keine Bewerbung, kein Lebenslauf, kein Zertifikat am Ende. Nur Code, der lebt.

Wie das hier funktioniert

Du forkst diese Seite. Du installierst Claude Code mit deinem eigenen Account. Du nimmst dir den ersten PR von der Liste unten und shippst ihn. Wenn er gemerged ist, der nächste. Nach vier PRs reden wir darüber, woran du als Nächstes arbeiten willst.

Keine Sessions, keine Calls, kein Curriculum. Du arbeitest, ich review, wir reden, wenn es Sinn ergibt.

Bevor du anfängst

  1. Lies die Philosophie. Wenn etwas nicht passt, sag es jetzt — das spart uns beiden Zeit.
  2. Hol dir Claude Code: npm install -g @anthropic-ai/claude-code, dann claude im Repo.
  3. Forke das Repo auf GitLab.
  4. Lies CLAUDE.md im Repo-Root. Das ist die Bedienungsanleitung.

Die ersten vier PRs

In dieser Reihenfolge. Nicht PR #2 anfangen, bevor PR #1 gemerged ist.

PR #1 — Tool-Karte zu „Built With"

Such dir ein Tool, das du wirklich benutzt. Füg eine tool-card in index.html ein. Trag den i18n-Key in alle 17 Sprachblöcke in i18n.js ein. Hol das offizielle Logo. Tests grün. Push.

Lehrt: HTML, i18n, Asset-Sourcing, das Test-Loop. Der minimale volle Zyklus.

PR #2 — Eine Support-Karte

Künstler, lokales Geschäft, oder eine Sache, die dir wichtig ist. Nur eine. Mit Fußnote, wenn du externe Fakten zitierst.

Lehrt: Fußnoten-System. Voice und Ton.

PR #3 — Eine Sprache, die du sprichst

Such dir eine Sprache aus i18n.js, die du wirklich kannst. Ersetze die EN-Fallbacks durch echte Übersetzungen. Nur eine Sprache. Nicht raten.

Lehrt: Scope-Disziplin. Großer sichtbarer Win.

PR #4 — Ein Blogpost (DE+EN)

Ein Thema, zu dem du wirklich was zu sagen hast. Zwei HTML-Files, Eintrag in posts.json, Feed regenerieren. Wenn du Fakten behauptest, check sie gegen Primärquellen.

Lehrt: End-to-end-Ownership.

Was „ready" heißt

  • Tests sind grün (cd tests && npx jest)
  • Du kannst in einem Satz sagen, was sich geändert hat und warum
  • Du hast deinen eigenen Diff gelesen, bevor du gepusht hast
  • Wenn UI: du hast es im Browser bei 375px und 1280px geöffnet

Das war's. Keine Checklisten-Theatralik.

Wenn du steckst

  • <10 Min stuck: mach weiter.
  • 10–30 Min: frag Claude anders. Formulier das Problem neu.
  • >30 Min: schreib mir. Was wolltest du, was hast du versucht, was passierte. Keine Entschuldigung. So ist der Deal.
